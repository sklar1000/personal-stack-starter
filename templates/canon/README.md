# Canon Starter Kit

Templates for building a governed knowledge canon.

## What's Here

| File | Purpose |
|------|---------|
| `Canon-README-template.md` | Root README for your canon |
| `Ontology-template.md` | Core concepts and definitions |
| `Framework-template.md` | Mental model documentation |
| `Governance-template.md` | Rules and change policies |
| `Example-template.md` | Worked example format |

## Setup

1. Create canon folder: `03_Knowledge/[Your-Domain]-Canon/`
2. Create subfolders:
   ```
   [Your-Domain]-Canon/
   ├── 01-ontology/
   ├── 03-frameworks/
   ├── 05-templates/
   ├── 06-examples/
   ├── 07-governance/
   └── README.md
   ```
3. Copy `Canon-README-template.md` → `README.md`
4. Copy `Ontology-template.md` → `01-ontology/[domain]-ontology.md`
5. Copy `Governance-template.md` → `07-governance/canon-rules.md`
6. Use `Framework-template.md` for each framework you document
7. Use `Example-template.md` for worked examples

## Build Order

1. **Ontology first** - Define core concepts before frameworks
2. **Governance early** - Establish rules before you have lots of content
3. **Frameworks next** - Document your mental models
4. **Examples always** - Every framework needs worked examples
5. **Templates last** - Only create what you'll actually use
